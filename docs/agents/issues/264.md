# Issue #264 — Consolidate duplicate beforeEach/afterEach setup in ActionEnqueuer and ActionsEnqueuer specs

**GitHub:** https://github.com/darthjee/navi/issues/264
**Label:** enhancement
**Impact:** Low | **Effort:** Low

## Problem

`ActionEnqueuer_spec.js` and `ActionsEnqueuer_spec.js` share nearly identical setup:

```js
beforeEach(() => {
  action = jasmine.createSpyObj('action', ['execute']);
  JobRegistry.build({ cooldown: -1 });
  spyOn(JobRegistry, 'enqueue').and.stub();
});
afterEach(() => {
  JobRegistry.reset();
});
```

The tiny differences between the two files are obscured by the repeated boilerplate, making it unclear what each spec is actually testing differently.

## Affected files

- `source/spec/lib/models/ActionEnqueuer_spec.js`
- `source/spec/lib/models/ActionsEnqueuer_spec.js`

## Proposed change

Extract the shared setup into a support context (e.g. `spec/support/contexts/actionEnqueuerContext.js`). Both spec files import and invoke the shared context, keeping only the test cases and any truly spec-specific setup local to each file.
