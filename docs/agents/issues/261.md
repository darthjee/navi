# Issue #261 — Reduce deep nesting in ResourceRequestJob_spec.js #perform block

**GitHub:** https://github.com/darthjee/navi/issues/261
**Label:** enhancement
**Impact:** Medium | **Effort:** Medium

## Problem

The `#perform` describe block in `ResourceRequestJob_spec.js` reaches 4 levels of nesting and duplicates spy setup across sibling `beforeEach` blocks. For example, both the "successful" and "failing" branches independently spy on `axios.get`, `Logger.info`, and `resourceRequest.enqueueActions`:

```js
// successful case (lines 42-47)
beforeEach(() => {
  response = { status: 200, data: '[]' };
  spyOn(axios, 'get').and.returnValue(Promise.resolve(response));
  spyOn(Logger, 'info').and.stub();
  spyOn(resourceRequest, 'enqueueActions').and.stub();
});

// failing case (lines 82-89) — nearly identical
beforeEach(() => {
  response = { status: 502, data: '[]' };
  spyOn(axios, 'get').and.returnValue(Promise.resolve(response));
  spyOn(Logger, 'error').and.stub();
  spyOn(Logger, 'info').and.stub();
  spyOn(resourceRequest, 'enqueueActions').and.stub();
});
```

This raises the cognitive cost of reading or changing the tests and makes it hard to know which setup applies to which expectation.

## Affected files

- `source/spec/lib/models/ResourceRequestJob_spec.js`

## Proposed change

- Flatten the nested `describe` blocks where the nesting does not add clarity.
- Extract repeated spy calls into a single top-level `beforeEach`, parameterising only what differs between cases (e.g. `status`, `data`).
- Aim for no more than 3 levels of nesting in any `describe` block.
