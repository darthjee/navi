# Issue #260 — Extract shared console/Logger spy setup into a test helper

**GitHub:** https://github.com/darthjee/navi/issues/260
**Label:** enhancement
**Impact:** Medium | **Effort:** Low

## Problem

Multiple spec files repeat the same pattern of spying on `console` or `Logger` methods:

```js
spyOn(console, 'debug').and.stub();
spyOn(console, 'info').and.stub();
spyOn(console, 'warn').and.stub();
spyOn(console, 'error').and.stub();
```

This copy-pasted block appears across several files. When the logging infrastructure changes, every file must be updated individually.

## Affected files

- `source/spec/lib/utils/logging/Logger_spec.js`
- `source/spec/lib/utils/logging/BaseLogger_spec.js`
- `source/spec/lib/models/Worker_spec.js`
- `source/spec/lib/models/ResourceRequestJob_spec.js`
- `source/spec/lib/services/Client_spec.js`

## Proposed change

Create a test support helper (e.g. `spec/support/helpers/loggerHelpers.js`) that exports a `stubAllLoggers()` function. Each affected file replaces its repeated spy block with a single call to that helper inside `beforeEach`.
