# Issue #265 — Introduce a test factory for wrapper objects in PathResolver, ParametersMapper, and PathSegmentTraverser specs

**GitHub:** https://github.com/darthjee/navi/issues/265
**Label:** enhancement
**Impact:** Medium | **Effort:** Low

## Problem

Three spec files repeatedly construct the same plain wrapper object inline:

```js
const wrapper = {
  parsedBody: { id: 1, name: 'Electronics', kind_id: 42 },
  headers: { page: '3' },
};
```

This pattern is repeated 6+ times in `ParametersMapper_spec.js` and 4+ times in `PathResolver_spec.js`. Any structural change to the wrapper requires updating 15+ locations across multiple files.

## Affected files

- `source/spec/lib/models/ParametersMapper_spec.js`
- `source/spec/lib/models/PathResolver_spec.js`
- `source/spec/lib/models/PathSegmentTraverser_spec.js`

## Proposed change

Create `spec/support/factories/WrapperFactory.js` with a `build(overrides)` function that returns a wrapper with sensible defaults. Replace all inline wrapper constructions with `WrapperFactory.build({ ... })` calls, passing only the fields that differ per test case.
