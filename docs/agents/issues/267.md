# Issue #267 — Consolidate repeated afterEach registry-reset blocks to top-level describe

**GitHub:** https://github.com/darthjee/navi/issues/267
**Label:** enhancement
**Impact:** Low | **Effort:** Low

## Problem

Several spec files repeat `afterEach(() => { SomeRegistry.reset(); })` inside multiple nested `describe` blocks instead of once at the top level. For example, `Config_spec.js` has three separate `afterEach` blocks all calling `ResourceRegistry.reset()` at different nesting levels. This creates confusion about cleanup order and requires updating multiple places when a new registry is introduced.

## Affected files

- `source/spec/lib/models/Config_spec.js`
- `source/spec/lib/models/ResourceRequestAction_spec.js`
- `source/spec/lib/server/Router_spec.js`
- `source/spec/lib/server/WebServer_spec.js`

## Proposed change

Move all registry-reset `afterEach` calls to the outermost `describe` block in each file. Where a reset is needed for multiple registries, consolidate them into a single `afterEach` that resets all of them. Verify that no test implicitly depended on mid-nesting cleanup by running the full suite after the change.
