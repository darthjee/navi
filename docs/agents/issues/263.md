# Issue #263 — Deduplicate axios mock-response pattern in ResourceRequestJob and Client specs

**GitHub:** https://github.com/darthjee/navi/issues/263
**Label:** enhancement
**Impact:** Medium | **Effort:** Low

## Problem

Both files repeatedly construct an axios mock response and stub `axios.get` with the same inline pattern:

```js
response = { status: 200, data: '[]' };
spyOn(axios, 'get').and.returnValue(Promise.resolve(response));
```

This appears 4+ times per file. The two files also use slightly different syntax (direct resolve vs intermediate variable), adding inconsistency on top of the duplication.

## Affected files

- `source/spec/lib/models/ResourceRequestJob_spec.js`
- `source/spec/lib/services/Client_spec.js`

## Proposed change

Create a test factory helper (e.g. `spec/support/helpers/axiosHelpers.js`) with functions like `stubAxiosGet(status, data)` and `stubAxiosGetFailure(error)`. Replace all in-line mock-response construction with calls to these helpers, enforcing a single, consistent pattern across both files.
